⚙️ Setup Instructions

Quick Start

  1. Enter Credentials: Add your SF Rec & Parks account username and password above
  2. Start Bot: Click "Start Bot" to begin monitoring every 5 minutes
  3. Manual Test: Use "Try Now" to test the reservation system immediately

Note: The bot is currently running with in-memory storage. Your data will be lost when the server restarts. For persistent storage, set up Supabase using the Database Setup tab.

Supabase Database Setup (Optional)

  1. Create Supabase Project: Sign up at supabase.com
  2. Get API Keys: Copy your project URL and anon key from Settings > API
  3. Set Environment Variables: 
    SUPABASE_URL=your_project_url
SUPABASE_ANON_KEY=your_anon_key
  4. Create Tables: Run this SQL in the Supabase SQL editor:
    Click to view SQL commands 
    -- User credentials table
CREATE TABLE user_credentials (
    id INTEGER PRIMARY KEY,
    username TEXT NOT NULL,
    password TEXT NOT NULL,
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Reservation logs table
CREATE TABLE reservation_logs (
    id SERIAL PRIMARY KEY,
    message TEXT NOT NULL,
    success BOOLEAN DEFAULT FALSE,
    details TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
  5. Restart: Restart your application to use Supabase storage
